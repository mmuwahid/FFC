§3.13 APPROVED · 20/APR/2026 (cross-ref design spec §3.13)

  • O1 · Tiebreak substitute → APPROVED. Phase 1 chain: points DESC → wins DESC → motms DESC → goals DESC → display_name ASC.
  • O2 · "Not yet played" group → KEEP AS SEPARATE GROUP. Muted bottom group, alpha-sorted, un-numbered.
  • O3 · Sort persistence → PERSISTENT. Writes to profiles.leaderboard_sort. New enum added to §2.1 + §2.2.
  • O4 · Medal icons → APPROVED. 🥇🥈🥉 top 3 of current season; trophies for archived.
  • S006 review additions:
    • W-D-L colour triplet — green W / grey D / red L applied to every W-D-L rendering across the app (row data AND column header).
    • Separators dropped — no dashes between W/D/L letters (header) or digits (rows). Clean flex gap instead.
    • Column header row added above ranked rows: Player · W D L · MP · Pts. Header letters use the same monospace font + column widths as the row digits so each letter sits directly above its number.
    • MP (matches played) column added right of W-D-L. Makes the W-D-L denominator explicit. Computed as wins + draws + losses.
    • Fixed column widths for W-D-L (56 px) / MP (28 px) / Pts (40 px) — numbers stay on the same horizontal line across all rows, row-to-row alignment is stable.
    • Medal tint on top 3 — faded gold / silver / bronze row fill + saturated border + inner highlight. Clearly differentiates podium rows from ranks 4–14 without overpowering the row content.
    • Late-cancel penalty column REMOVED from the leaderboard (previously mocked in tile 4/5/6). Penalties still deduct points in the standings calculation, but the `−1` / `−2` indicator now surfaces on the post-match report (§3.15) instead, so player names retain breathing room on the leaderboard rows.

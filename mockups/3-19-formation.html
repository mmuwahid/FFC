State C · Auto-assigning rotation numbers 1–6

Deterministic, captain-overridable

When the captain picks the starting GK, the remaining 6 players are auto-numbered 1–6 by default order: DEF → CDM → W → ST, ties broken by season leaderboard rank (stronger player = earlier number so they rotate into goal sooner while the game is still close). Captain can long-press any rotation chip and drag to reorder. Order is persisted in formations.rotation_order (int[]).

GK · OK then rotates → 1 · SM 2 · RP 3 · BK 4 · AR 5 · TH 6 · FM
Minute 0–10: OK in goal. 10–20: SM swaps in, OK returns to DEF. 20–30: RP. And so on. Game pauses briefly at each rotation tick — in practice captains align swaps with natural stoppages.
State D · Realtime share & update

Non-captains see every edit within 2 seconds

Supabase realtime subscription on formations + formation_rotation_order. Tokens animate to new positions in 200ms; roster rotation chips cross-fade. Re-shares after the first are throttled — max one push notification per viewer per 60s. Offline viewers see a "Last synced HH:mm" chip until reconnect; formation stays cached locally.

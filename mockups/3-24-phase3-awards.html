Behavior summary

  • Active season shows hero cards as provisional — re-computes live from match_players + matches on each load.
  • Ended season shows hero cards from a frozen snapshot (season_awards table). "Provisional" badge drops; "Active" badge becomes "Final".
  • Season pill lets user view awards for any season — past or current. Each season has its own awards page.
  • Wall of Fame reads only from season_awards snapshot table → only shows ended seasons (active season already covered above).
  • Tie-breaking follows existing Leaderboard sort cascade (most wins → fewest cards → alphabetical) for all 3 awards. Single deterministic winner.
  • Snapshot trigger: when admin ends a season (existing flow), AFTER UPDATE trigger inserts 3 winner rows into season_awards. Deterministic; reuses existing season-end flow; no new admin action.
  • Hero name + Wall of Fame name cells are tappable — navigate to that player's profile filtered to that season.
  • Empty Wall of Fame (first-ever season): shows "First season — Wall of Fame begins after Season 11 ends" placeholder.
  • Empty active season (0 matches yet): hero cards show "No matches played yet" with greyed trophy + "—" name.

# FFC Lessons Learned

FFC inherits **all lessons** from `Padel Battle APP/tasks/lessons.md` — read that before writing any FFC code. Every Critical Rule there applies here too.

## Critical Rules (Always Enforce — inherited from PadelHub)
1. Mockup-first workflow. HTML mockup → user review → finalize → implement.
2. No design changes without explicit approval.
3. Placeholder data only in mockups. No invented names / leagues / stats.
4. Version control planning docs. V1, V2, V3. Never overwrite.
5. Never regenerate user documents. Edit originals.
6. Match existing app styling in mockups. Copy CSS from most recent approved mockup.
7. Safe-area insets for all fixed-position elements on mobile.
8. Verify before pushing fixes: run dev server, check console, visually verify.
9. When adding inline state-driven UI, declare `useState` first. Grep to confirm.
10. CRITICAL: verify DB columns exist before any Supabase `.select()` / `.update()`.
11. Never use `useMemo` for React Context with derived state. Plain object.
12. Never place hooks after conditional returns.
13. Global `<style>` must live in main render, not early-return subtree.
14. SECURITY DEFINER RPCs for all Edge Function DB access (never service-role + direct queries).
15. RPC parameters receiving JSON should be typed `TEXT`, cast to `JSONB` internally.
16. After `CREATE TABLE` via SQL editor: `GRANT SELECT, INSERT, UPDATE, DELETE ON public.X TO authenticated, anon;`
17. Always include `window.location.search` in `redirectTo` URLs (OAuth + reset).
18. Always scope `.delete()` / `.update()` with `.eq("league_id", ...)` — defense-in-depth beyond RLS.
19. Bump service-worker `CACHE_NAME` on every major deploy. Format: `ffc-vN`.
20. Full Windows path in Node scripts: `C:/Users/User/AppData/Local/Temp/FFC/...` — never `/tmp/`.
21. CRLF-aware string replacements on Windows git clones.
22. After triggering React state changes in `preview_eval`, wrap DOM reads in a 200–300ms timeout.
23. Git identity for FFC commits: `git -c user.name="Mohammed Muwahid" -c user.email="m.muwahid@gmail.com"`.

      $bytes = $_.Export([Security.Cryptography.X509Certificates.X509ContentType]::Cert)

## Patterns That Work (S028)

- [23/APR/2026] (S028) **Extract coordinate-heavy data to a sibling module.** Formation presets (170 LOC of x/y coordinates across 9 patterns) lived in `src/lib/formationPresets.ts`, keeping `FormationPlanner.tsx` focused on state + render. Pattern helpers (`presetsForFormat`, `getPreset`, `rosterSizeForFormat`) + the `CUSTOM_PATTERN` sentinel colocated with the data. **Why worth remembering:** data-in-page would have inflated FormationPlanner.tsx past 800 LOC; staying under 600 made the drag/GK/rotation slices land clean. General rule: when a UI component needs a large table of constants, move it to a `src/lib/<domain>Presets.ts` with typed helpers.
- [23/APR/2026] (S028) **Ref-based drag state — not `useState`.** `dragStateRef: { slotIdx, pointerId } | null` is mutated at every `pointermove`. If this were `useState`, every pointer move would trigger a React re-render cycle; ref keeps renders limited to the actual `setLiveSlots` update. **Why worth remembering:** any drag-handler that mutates per-frame should use a ref for the capture state, not state. React's reconciler is plenty fast for position updates via setState, but pointermove fires ~60 times per second — gating that on a ref check (`if (!state \|\| state.pointerId !== e.pointerId) return`) is cheaper than a render.
- [23/APR/2026] (S028) **Slice-size discipline with deferral notes in the file header.** §3.19 Formation split into A/B/C/D/E; each slice's file-header JSDoc lists what lands + what's deferred to later slices. Progress is legible at a glance; PRs stay reviewable. **Why worth remembering:** the alternative — cramming all 5 slices into one commit — breaks code review and loses the "what ships next?" signal. For any screen with multi-session scope, split AND document the split in-code.
- [23/APR/2026] (S028) **Conditional spread for optional RPC args.** `...(gkProfileId ? { p_starting_gk_profile_id: gkProfileId } : {})` produces exactly the right runtime JSON (key absent vs key: null) and exactly matches the generated type's `T \| undefined` signature — no `as`-cast escape hatch needed. **Why worth remembering:** pattern for every Supabase RPC with default-null optional args. Cleaner than `p_field: gkProfileId ?? undefined` (which still has the type-check concern), cleaner than `as unknown as null` (which breaks strict-build).
- [23/APR/2026] (S028) **Auto-default on mode-toggle with sensible fallback chain.** When captain flips GK mode to "rotate" for the first time, the starting-GK auto-defaults to slot-0 player *if* they're a profile member, *else* to the first profile member on the team. Captain can override via the picker. **Why worth remembering:** forcing captains to re-pick from zero is bad UX; the fallback chain gets them to a valid state in one tap. Applies to any "mode switch opens new required field" flow — pre-fill with the most-likely-intended value.
